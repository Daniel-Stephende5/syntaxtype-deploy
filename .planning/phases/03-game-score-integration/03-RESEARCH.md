## Technical Implementation Strategy

### Backend Approach (Per CONTEXT.md)

**Unified Endpoint:** `POST /api/scores/{category}`
- Extends existing ScoreController
- Game type passed as path variable
- User identified from JWT token (secure, frontend doesn't pass userId)
- Saves to Score table AND updates Leaderboard if better

**Score Types:**
- Typing games (TYPING_TESTS, FALLING_WORDS): rank by combined score (WPM × accuracy/100 × 1.5 if accuracy > 95%)
- Non-typing games (GALAXY, GRID, BOOKWORM, CROSSWORD, FOUR_PICS, SYNTAX_SAVER): rank by raw score
- OVERALL_TYPING: best from typing games
- OVERALL_GAMES: best from non-typing games

**Update Logic:**
- Service layer checks if new score > existing best before DB write
- Only update Leaderboard when new score beats previous best
- Full history kept in Score table for analytics

### Frontend Changes Required

**TypingTest.js & FallingTypingTest.js:**
- Already have score submission, need to add Leaderboard category parameter and handle response

**GalaxyMainGame.js, GridGame.js, Bookworm.js, CrosswordGame.js, SyntaxSaverLesson.js:**

## Common Pitfalls

1. **Forgetting category parameter:** Each game must pass correct category
2. **Not handling auth:** Allow play without login, check JWT at submission time
3. **Race conditions:** Service should compare before save
4. **Missing WPM/accuracy for non-typing:** Send wpm: 0, accuracy: 100 for non-typing games
5. **Four Pics score tracking:** Must implement score tracking before submission
